# UPS Shipping

`description: J2Commerce (formerly known as J2Store)`

This plugin integrates the UPS Shipping Rate API with J2Commerce.

## Requirements

- with PHP 8.3.0 +
- Joomla! 6.x
- J2Commerce 6.x**‌**

## Configuration

Prior to configuring the plugin, you should have an active account with UPS and have received API keys. Contact UPS to register and get API keys.

## **‌**Purchase and Download

You can install this USP shipping plugin using Joomla installer. The following steps help you with a successful installation.

‌**Step 1:** Go to our [J2Commerce website](https://www.j2commerce.com/) > Extensions > Shipping Plugin

**Step 2:** Locate the UPS App > click View Details > Add to cart > Checkout.&#x20;

**Step 3:** Go to your My Downloads under your profile button at the top right corner and search for the app. Click Available Versions > View Files > Download Now

## Installation

You can install this UPS shipping plugin using Joomla installer. The following steps help you with a successful installation.

**Step 1:** In the Joomla admin, go to System > Install > Extensions

![Install extensions](<../../assets/app install1 (1) (1).webp>)

**Step 2**: Click on the Browse button and select the UPS Shipping zip file you saved during the download process. You will receive a message of success once installed.

![app install](../../assets/ups-download.webp)

## Enable&#x20;

Once you have installed the extension, you will need to enable it. There are **two** ways you can access the extension.&#x20;

**a:** Go to the **J2Commerce** icon at the top right corner **-> Setup -> Shipping Methods**

**b:** Go to **Components** on the left sidebar **-> J2Commerce -> Dashboard -> Setup -> Shipping Methods**

![shipping method](/img/shipping-methods.webp)

Look for UPS Shipping, click the **X,** and it will turn into a green checkmark. It is now enabled and ready for setup.

![](/img/shipping-methods-enable.webp)

## Configure the Plugin

Once you click on the **UPS Shipping** Title next to the green checkmark, you are ready to start setting up the UPS parameters.

**Helpful Tip:** Click on the Toggle Inline Help button on any app/plugin you install and it will show a description below each section. See image below

![](/img/ups-toggle.webp)

### Plugin Tab

![](/img/ups-setup.webp)

***\*\*Do you need to know how to obtain UPS OAuth Credentials? Go to the bottom of this document for the step-by-step details.***

**Production Client ID:** Enter the Client Key provided by the UPS Developer Portal ([https://developer.ups.com/](https://developer.ups.com/)).

**Production Client Secret:** Enter the Client Secret provided by the UPS Developer Portal  ([https://developer.ups.com/](https://developer.ups.com/)).

‌**Shipper Number:** Your UPS shipper number (account number). Required for negotiated rates.

**‌Rating Request Type:**&#x20;

- **Shop:** returns rates for all available services.&#x20;

- **Rate:** returns a rate for the single selected service.

**Enabled Services:** Select which UPS services to display at checkout. Only services returned by UPS for the shipment will be shown.

**UPS Packaging Box:** The UPS packaging type to use for all shipments.

**Packing Mode:**&#x20;

- **Per Item:**  Sends one package per item to UPS.&#x20;

- **Box Packing:** Uses the 3D bin packing algorithm to optimally pack items into configured boxes for more accurate rates.

**Weight Unit:** Unit of measurement for package weights sent to UPS.

**Dimension Unit:** Unit of measurement for package dimensions sent to UPS.

**Send Package Dimensions:** Include length, width, and height when requesting rates. Disable if your products do not have dimensions set.

![](/img/ups-setup1.webp)

**Default Weight:** Fallback weight used when a product has no weight configured.

**‌Default Length:** Fallback length used when a product has no length configured.

**Default Width:** Fallback width used when a product has no width configured.

**Default Height:** Fallback height used when a product has no height configured.

**Shipping Postal Code:** The postal or ZIP code of your warehouse or shipping location.

**Shipping Country:** Select the country of your shipping location.

**Shipping State:** Select the state or province of your shipping location.

**Shipping City:** City name of your shipping location.

**Shipping Origin Address Type:**  Set to **Yes** for residential delivery. When enabled, UPS applies residential surcharges. Set to **No** for commercial addresses.

**Enable Negotiated Rate:** Request negotiated (discounted) rates. Requires a valid shipper number and that your account has negotiated rates enabled.

**Pickup Type:** The UPS pickup type for this shipper location. Most stores use Daily Pickup.

**Delivery Confirmation Option:** Require delivery confirmation for all packages. Signature options add a surcharge to each shipment.

**Handling Cost:** Additional cost added to every UPS rate. Set to 0 for no handling cost.

**Handling Cost Type:** Whether the handling cost is a fixed amount or a percentage of the shipping rate.

**Tax Profile:** Select a tax profile to apply tax to shipping costs. Leave as None for no shipping tax.

**Geozone Restriction:** Restrict this shipping method to customers in a specific geozone. Leave as None to make available to all customers.

**Show Delivery Date:** Set to **Yes** to display estimated business days in transit next to each shipping option at checkout. See the images below.

![Delivery date examples](<../../assets/ups-perameters5 (2).webp>)

**Debug Mode:** Enable debug logging. API requests and responses are written to administrator/logs/shipping\_ups.php. Disable in production.

**Note:** In the production environment, it must be set to NO.

## How to obtain UPS OAuth Credentials:

The following instructions will help walk you through the process of obtaining your new UPS OAuth Client ID and Client Secret.&#x20;

**Step 1:** Go to ([https://developer.ups.com/](https://developer.ups.com/)) and log in.

**Step 2:** Click My Apps in the top right corner

**Step 3:** Click Add Apps in the bottom left corner

![Add apps](../../assets/ups-add-apps.webp)

**Step 4:** Then, I need API credentials because I want to integrate UPS technology into my business.

**Step 5:** On the same page as #4: Select the client's UPS account you need to link it to

![](../../assets/ups-add-account.webp)

**Step 6:** Complete the contact info

**Step 7:** Then select the Rating, Tracking, and Time In Transit API's (I believe the Authorization(OAuth) is automatically added when you save)

(Right now, we aren't using the Tracking API, but we do plan on incorporating that in later versions, so better to add it now so you don't need to worry about adding it later...)

![API's](../../assets/ups-api.webp)

**Step 8:** After saving, you should be able to see the Client ID and client Secret under the App Info/Credentials page

![](../../assets/ups-approval.webp)
